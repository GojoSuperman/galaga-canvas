# 갤러그 스타일 슈팅 게임 — 설계 문서

- 작성일: 2026-07-14
- 프로젝트: `/mnt/f/aiffel/html-game`
- 상태: 승인됨 (구현 계획 수립 대기)

## 1. 목표

브라우저에서 바로 실행되는 갤러그(Galaga) 스타일 2D 종스크롤 슈팅 게임. 빌드 도구와 외부 라이브러리 없이 순수 HTML + ES 모듈 + Canvas 2D로 구현한다. 원작의 핵심 재미(편대 진입 → 대형 유지 → 급강하 공격)를 재현하고, 파워업과 보스전을 더한 확장형으로 만든다.

### 성공 기준

- `index.html`을 로컬 정적 서버로 열면 즉시 플레이 가능하다.
- 5개 스테이지를 클리어해 엔딩(보스 격파)에 도달할 수 있다.
- 적이 곡선 경로로 진입하고, 대형을 이루고, 개별적으로 급강하 공격한다.
- 4종 파워업이 드롭·획득·소멸한다.
- 효과음·BGM이 오디오 파일 없이 재생된다.
- 최고 점수가 브라우저에 저장된다.
- 이미지 파일과 오디오 파일이 저장소에 0개다.

### 비목표 (YAGNI)

- 모바일 터치 조작, 반응형 레이아웃
- 멀티플레이, 서버, 리더보드 백엔드
- 원작의 보너스(챌린징) 스테이지
- 원작의 트랙터 빔 포획 / 듀얼 파이터 (파워업 무기 레벨업이 이 역할을 대신함)
- 빌드 파이프라인(번들러, 트랜스파일러, 타입스크립트)

## 3. 아키텍처

### 3.1 채택안: 클래스 엔티티 + 데이터 주도 스테이지

엔티티(Player, Enemy, Bullet, Powerup, Boss)는 `update(dt)` / `draw(ctx)`를 가진 일반 클래스로 두고, **스테이지 구성·적 배치·난이도 수치는 전부 `stages/stages.js`의 데이터**로 분리한다.

대안으로 ECS와 전면 하드코딩을 검토했으나,

- ECS는 엔티티 200개 미만 규모에서 보일러플레이트만 늘린다.
- 하드코딩은 밸런스를 만질 때마다 로직을 수정해야 한다.

갤러그는 적의 움직임 수치(적 수, 급강하 빈도, 탄속, 진입 경로)를 반복적으로 조정하게 되므로, 그 수치가 한 파일에 모여 있는 것이 결정적으로 유리하다.

### 3.3 렌더링

- 논리 해상도 **480 × 640** 고정. CSS로 정수배 확대하고 `image-rendering: pixelated` 적용 → 레트로 픽셀 감성 유지, 좌표 계산 단순화.
- 배경은 3레이어 스타필드(속도가 다른 별 3겹)를 코드로 그린다.
- 렌더 순서: 배경 → 파워업 → 적/보스 → 플레이어 → 탄 → 파티클 → HUD.

### 3.4 게임 루프

`requestAnimationFrame` + 누적자(accumulator) 기반 **고정 타임스텝 60Hz** 로직 업데이트. 프레임 드랍 시에도 물리·AI가 일정하게 동작한다. 렌더는 매 프레임 1회. 보간(interpolation)은 픽셀 스냅 스타일에서 이득이 없으므로 넣지 않는다.

## 4. 스프라이트 생성 (핵심 설계)

`gfx/pixels.js`에 각 스프라이트를 **문자열 배열 + 팔레트**로 정의한다.

```js
export const PALETTES = {
  player: { R: '#e33', W: '#fff', B: '#39f', '.': null },  // null = 투명
};

export const PLAYER_SHIP = [
  '..R..',
  '.RWR.',
  'RWBWR',
  'R.B.R',
];
```

`spriteFactory.js`가 부팅 시 이 데이터를 순회하며 오프스크린 `<canvas>`에 픽셀 하나당 N×N 사각형으로 그려 `HTMLCanvasElement`로 캐싱한다. 게임 루프에서는 `ctx.drawImage(cachedSprite, x, y)` 한 번만 호출한다.

**교체 가능성**: `spriteFactory`는 `getSprite(name) → CanvasImageSource`라는 인터페이스만 노출한다. 나중에 실제 PNG 스프라이트 시트로 바꾸고 싶으면 이 모듈 내부만 교체하면 되고, 게임 로직은 손대지 않는다.

필요한 스프라이트: 플레이어 기체, 적 3종(A/B/C), 보스, 플레이어 탄, 적 탄, 파워업 아이콘 4종, 폭발 4프레임, 목숨 표시용 미니 기체.

## 5. 적 AI — 갤러그의 핵심

각 적은 4상태 FSM을 갖는다.

| 상태 | 동작 | 전이 조건 |
|---|---|---|
| `ENTERING` | 화면 밖에서 베지어 곡선을 따라 비행 (루프·나선 등) | 경로 끝 도달 → `IN_FORMATION` |
| `IN_FORMATION` | 대형 그리드의 자기 슬롯에 정렬, 대형 전체가 좌우로 흔들림 | 스포너가 급강하 지명 → `DIVING` |
| `DIVING` | 플레이어를 향해 곡선 강하하며 사격 | 화면 아래 이탈 → `RETURNING` |
| `RETURNING` | 화면 위쪽에서 재진입해 자기 슬롯으로 복귀 | 슬롯 도달 → `IN_FORMATION` |

- **대형(`formation.js`)**: 열×행 그리드의 슬롯 좌표를 계산하고, 전체 대형을 사인파로 좌우 흔든다. 적은 자기 슬롯 좌표를 매 프레임 조회한다 → 대형 이동과 개체 로직이 분리된다.
- **경로(`paths.js`)**: 3차 베지어 제어점 세트를 이름으로 정의(`ENTRY_LOOP_LEFT`, `DIVE_SWOOP` 등). 경로 추종은 매개변수 t를 시간에 따라 진행시켜 좌표를 얻는다.
- **급강하 지명**: `playScene`이 스테이지 데이터의 `diveInterval`마다 대형 내 적을 무작위로 1~N기 골라 `DIVING`으로 전이시킨다.

**점수**: 대형 안의 적을 잡으면 기본 점수, **급강하 중인 적을 잡으면 2배**. (원작의 리스크-리워드 규칙)

## 6. 파워업

적 격파 시 확률(스테이지 데이터의 `dropRate`, 기본 10%)로 드롭. 아래로 낙하하며, 플레이어와 접촉 시 획득. 화면 아래로 나가면 소멸.

| 파워업 | 효과 | 지속 |
|---|---|---|
| 무기 레벨업 (W) | 싱글 → 더블 → 트리플(3방향 확산). 최대 3 | **영구** (사망 시 1레벨 감소, 최소 1) |
| 실드 (S) | 피격 1회를 무효화, 기체 주위에 링 표시 | **충전형** (1회 소모) |
| 연사 (R) | 발사 쿨다운 ×0.5 | **10초** |
| 속도 (P) | 이동 속도 ×1.4 | **10초** |

**사망 시**: 목숨 -1, 무기 레벨 -1, 실드 해제, 시한 버프 전부 해제, 2초 무적(깜빡임) 후 재개.

## 7. 스테이지 & 보스

`stages/stages.js`가 스테이지 배열을 내보낸다. 각 스테이지는 다음을 갖는다.

```js
{
  waves: [ { enemyType, count, entryPath, formationRow } , ... ],
  diveInterval: 2.0,      // 초 — 급강하 지명 간격
  diveCount: 1,           // 한 번에 급강하할 적 수
  enemyBulletSpeed: 180,  // px/s
  dropRate: 0.1,
}
```

- **스테이지 1~4**: 적 편대 구성이 점점 커지고, `diveInterval`이 짧아지며, `diveCount`·`enemyBulletSpeed`가 오른다. 편대를 전멸시키면 클리어.
- **스테이지 5 (보스)**: 대형 보스 1기 + 호위 적. 보스는 HP 게이지를 갖고 화면 상단을 좌우로 이동한다.

**보스 3페이즈** (HP 구간으로 전이):

| 페이즈 | HP | 패턴 |
|---|---|---|
| 1 | 100% ~ 66% | 확산탄 (부채꼴 5발) |
| 2 | 66% ~ 33% | 확산탄 + 졸개 주기적 소환 |
| 3 | 33% ~ 0% | 이동 속도 상승 + 플레이어 조준 탄막 연사 |

보스 격파 시 엔딩(클리어 화면 → 게임오버 씬에서 최종 점수·하이스코어 표시).

## 8. 오디오

오디오 파일 없이 **WebAudio API로 전부 합성**한다.

- **효과음**: 발사(짧은 하강 스퀘어 blip), 적 폭발(노이즈 버스트 + 로우패스), 플레이어 사망(하강 소우), 파워업 획득(상승 아르페지오), 보스 피격(짧은 노이즈), 스테이지 클리어(상승 3음).
- **BGM**: 스퀘어 리드 + 트라이앵글 베이스의 8비트풍 루프. lookahead 스케줄러(25ms 간격으로 100ms 앞을 예약)로 타이밍 흔들림을 막는다.
- 브라우저 자동재생 정책 때문에 **최초 키 입력 시 `AudioContext`를 resume**한다.
- `M` 키로 전체 음소거 토글.

## 9. 화면 흐름 & 조작

```
타이틀 ──Space──▶ 플레이 ──목숨 0──▶ 게임오버 ──Space──▶ 타이틀
                    │  ▲                  ▲
                    │  │ P                │
                    └──┴─ 일시정지        └── 보스 격파(클리어)
```

| 키 | 동작 |
|---|---|
| ← → | 좌우 이동 |
| Space | 발사 / 화면 전환 |
| P | 일시정지 |
| M | 음소거 토글 |

**HUD**: 좌상단 점수, 우상단 최고점수, 좌하단 목숨(미니 기체 아이콘), 우하단 스테이지 번호 + 무기 레벨 + 활성 버프 타이머 바.

**하이스코어**: `localStorage`에 TOP 5 점수 배열을 저장하고 타이틀·게임오버 화면에 표시한다. 저장소 접근은 `core/storage.js`에서 try/catch로 감싸, 프라이빗 모드 등으로 실패해도 게임이 죽지 않게 한다.

## 10. 충돌 처리

AABB(축 정렬 사각형) 판정. 엔티티가 200개 미만이므로 공간 분할 없이 전수 검사로 충분하다.

검사 쌍:
1. 플레이어 탄 ↔ 적 / 보스
2. 적 탄 ↔ 플레이어
3. 적(급강하 중) ↔ 플레이어 (몸통 박치기)
4. 파워업 ↔ 플레이어

`collision.js`는 상태를 갖지 않는 **순수 함수**(`aabbOverlap(a, b) → boolean`)만 노출한다 → 단위 테스트가 쉽다.

## 11. 성능

- 탄·파티클은 **오브젝트 풀링**으로 재사용해 GC 스파이크를 막는다.
- 스프라이트는 부팅 시 1회 생성 후 캐싱 — 프레임마다 픽셀을 다시 그리지 않는다.
- 목표: 1920×1080 노트북 브라우저에서 안정적 60fps.

## 12. 검증 전략

**단위 테스트 (`node:test`, 의존성 0)** — 순수 로직만:

- `collision.js` — AABB 겹침/비겹침/모서리 접촉
- `formation.js` — 슬롯 좌표 계산, 흔들림 오프셋 경계
- `paths.js` — 베지어 t=0, t=1이 시작·끝 제어점과 일치
- 점수 계산 — 대형 적 vs 급강하 적 2배 규칙
- 파워업 상태 — 무기 레벨 상한(3), 사망 시 감소, 버프 타이머 만료
- `storage.js` — TOP 5 정렬·절단, 저장소 실패 시 폴백

**수동 검증 (브라우저)** — 플레이 감각과 렌더링:

로컬 정적 서버(`python3 -m http.server`)로 띄우고 실제 플레이해 확인한다. 자동화하기 어렵고 자동화 가치도 낮은 항목: 적 진입 곡선이 자연스러운지, 조작감이 무거운지, 난이도 곡선이 적절한지, 사운드가 거슬리지 않는지.

## 13. 위험 요소

| 위험 | 대응 |
|---|---|
| 픽셀아트를 코드로 짜면 모양이 조악할 수 있음 | 픽셀맵을 `pixels.js` 한 곳에 모아 즉시 수정 가능하게 함. 초기 구현 후 사용자 확인 |
| WebAudio BGM이 거슬릴 수 있음 | 기본 볼륨을 낮게(0.3) 잡고 `M` 음소거 제공. 효과음과 BGM 볼륨 분리 |
| 베지어 경로 튜닝이 오래 걸림 | 경로를 `paths.js`에 이름으로 분리해 스테이지 데이터에서 교체 가능하게 함 |
| 난이도 밸런스 | 모든 수치를 `stages.js`에 모아 한 파일에서 조정 |
